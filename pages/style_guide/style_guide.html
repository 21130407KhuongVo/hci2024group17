Style Guide for Team 17

This style guide outlines the design principles and components to be used in our project, leveraging Bootstrap 5.3 for a consistent and responsive layout.

Color Palette

  • Primary Color: #007bff (Bootstrap Primary)
  • Secondary Color: #6c757d (Bootstrap Secondary)
  • Success Color: #28a745 (Bootstrap Success)
  • Danger Color: #dc3545 (Bootstrap Danger)
  • Warning Color: #ffc107 (Bootstrap Warning)
  • Info Color: #17a2b8 (Bootstrap Info)

Typography

  • Headings: Use Bootstrap's heading classes (h1, h2, h3, etc.) for semantic structure.
  • Body Text: Use class="text-body" for standard body text.
  • Links: Use class="link-primary" for primary links.

Buttons

Utilize Bootstrap button classes for consistent styling:

  • class="btn btn-primary" for primary buttons.
  • class="btn btn-secondary" for secondary buttons.
  • class="btn btn-success" for success buttons.

Forms

Use Bootstrap form classes to ensure proper spacing and alignment:

  • class="form-control" for input fields.
  • class="form-check" for checkboxes and radio buttons.

JavaScript Components

Incorporate jQuery for dynamic interactions:

  • Use $(document).ready(function() { ... }); to ensure scripts run after the DOM is fully loaded.
  • Utilize Bootstrap's JavaScript components like modals and dropdowns for enhanced user experience.

Responsive Design

Ensure all components are responsive by using Bootstrap's grid system and utility classes.

Project Directory Structure

The following is the directory structure of our project, which is organized to facilitate easy navigation and maintainability:


.
├── assets
│   ├── images          # Contains all image assets used in the project
│   │   ├── covers      # Cover images for playlists or albums
│   │   ├── members     # Profile images of team members
│   │   ├── nlu         # Logos and branding for NLU
│   │   └── requirements # Screenshots and design requirements
│   ├── medias          # Audio files for the application
│   └── styles          # CSS files for styling the application
├── components          # Reusable HTML components
│   ├── footer.html     # Footer component
│   ├── header.html     # Header component
│   └── sidebar.html    # Sidebar component
├── data                # JSON data files
│   └── information.json # Contains information used in the application
├── index.html          # Main entry point of the application
├── main.js             # Main JavaScript file for application logic
├── pages               # Contains different pages of the application
│   ├── SignIn.html      # Sign In page
│   └── style_guide      # Style guide page
│       ├── style_guide.html # HTML for Style Guide
│       └── style_guide.js   # JavaScript for Style Guide
└── README.md            # Project documentation

This structure helps in organizing the project efficiently, making it easier for team members to locate files and understand the project layout.
